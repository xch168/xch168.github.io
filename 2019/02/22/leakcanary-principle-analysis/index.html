概述

LeakCanary是一个开源的内存泄漏检测库，极大简化了内存泄漏的检测流程。了解其工作原理，有助于我们更好的理解Android的内存管理机制。

工作机制

  1. RefWatcher.watch()创建一个KeyedWeakReference到要被监控的对象。
  2. 然后在后台线程检查引用是否被清除，如果没有，调用GC。
  3. 如果引用还是未被清除，把heap内存dump到APP对应的文件系统中的一个.hprof文件中。
  4. 在另一个进程中的HeapAnalyzerService有一个HeapAnalyzer使用HAHA解析这个文件。
  5. 在Heap Dump中，HeapAnalyzer根据唯一的reference key找到了KeyedWeakReference，并定位了泄漏的引用。
  6. HeapAnalyzer计算到GC Roots的最短强引用路径，并确定是否泄漏，如果是的话，建立导致泄漏的引用链。
  7. 引用链传递到APP进程中的DisplayLeakService，并以通知的形式展示出来。
